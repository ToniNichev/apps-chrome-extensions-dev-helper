+ Setting up asset rewrite rules
Creating asset rewrite rule will allow chrome to dynamically replace matching url with Replacement URL
For Example if you want to test and customize Java Script, you could set up rewrite rule to loast http://mysite.com/script_B.js instead of http://mysite.com/script_A.js
You could also override and add request and response headers.
x active global case insensitive
Match URL Matching url is regular expression
Leaving it empty, will match everything

Example:
http://cnn.com/.* - this will match everything after the third slash (.*) http://cnn.com/news, or http://cnn.com/live
Replacement URL Replacement URL that points to the new asset that will be loaded.


Example:
http://mysite.com/script.js

Request Headers Request Headers override
Put each header override on new line. Header name and value are separated by colon.
Example:
Host:mywebsite.com
user-agent:Apple-iPhone2C1/801.306

Response Headers Response Headers override
Put each header override on new line. Header name and value are separated by colon.
Response headers will be overriten, but it won't be shown in Crome console.

Example below will allow cross domain script loading.
Access-Control-Allow-Origin:*

+ Each following proxy Rule overrides the previous one
This means that if you have two proxy rules that match the same url pattern (or emty url pattern which matches all), each following proxy rule will override the previous one.
